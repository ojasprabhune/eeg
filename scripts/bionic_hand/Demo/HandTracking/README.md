# Hand Tracking Node

This node provides hand tracking capabilities using MediaPipe to control the bionic hand. It supports both real-time webcam tracking and playback from preprocessed joint data.

## Scripts

- `main.py`: Opens the webcam and tracks hand keypoints in real-time. This is the default node for live interaction.
- `data_control.py`: Controls the hand using pre-recorded `.npy` files generated by `scripts/data/collect_data.py`.

## Usage

### Webcam Tracking
To run the webcam tracking node independently (requires a running dora daemon):
```bash
uv run HandTracking/main.py
```

### Data Playback
To control the hand using a saved `.npy` file:
```bash
uv run HandTracking/data_control.py --data_file path/to/your_data.npy
```

## Configuration

### Platform Port Settings
By default, the system is configured for macOS using `/dev/ttyAMC0`. 

If you are using **Windows**, you must identify your serial port (e.g., `COM13`) and update the `hand_controller` node arguments in the following files:
- `scripts/bionic_hand/Demo/dataflow_real.yml`
- `scripts/bionic_hand/Demo/dataflow_tracking_real.yml`

Change:
`args: --serialport /dev/ttyAMC0 ...`
To:
`args: --serialport COM13 ...`
